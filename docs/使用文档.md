# NetworkTest 使用文档

`networktest.sh` 是一个在 Linux VPS 上运行的一体化网络检测脚本，用于回答一个核心问题：**"我这台 VPS 是通过什么线路接入中国大陆的？质量如何？"**

包含四大功能，通过菜单选择或命令行参数调用：

| # | 功能 | 说明 |
|---|---|---|
| 1 | 回程线路检测 | 判断到三网的回程走 CN2 GIA / CN2 GT / 163 / 9929 / 4837 / CMI / CMIN2 中的哪条 |
| 2 | 全国延迟测试 | 对全国 12 个主要城市 × 三网共 36 个节点并发 ping，输出延迟/丢包并按运营商汇总 |
| 3 | 全国测速 | 对大陆三网 Speedtest 节点做上传/下载测速 |
| 4 | 全球测速 | 对亚太/欧洲/北美等全球主要城市做上传/下载测速 |

---

## 1. 快速开始

### 方式一：下载后运行（推荐）

```bash
curl -fsSL -o networktest.sh https://raw.githubusercontent.com/Yangdongle668/NetworkTest/main/networktest.sh
chmod +x networktest.sh
sudo ./networktest.sh
```

### 方式二：一键运行

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Yangdongle668/NetworkTest/main/networktest.sh)
```

> 建议以 **root** 运行：脚本需要自动安装依赖（curl / jq / ping / traceroute），并使用 ICMP 做路由追踪。非 root 也能运行，但缺失的依赖需要手动安装。

运行后进入主菜单：

```
========== NetworkTest v1.0.0 主菜单 ==========
 1. 回程线路检测   (判断接入大陆的线路类型)
 2. 全国延迟测试   (三网各省市 ping 延迟/丢包)
 3. 全国测速       (大陆三网上传/下载测速)
 4. 全球测速       (全球主要地区上传/下载测速)
 5. 全面测试       (依次执行 1-4)
 0. 退出
```

输入编号执行对应功能，执行完自动返回菜单，可连续测试多个项目，输入 `0` 退出。

## 2. 命令行参数（免交互，适合自动化）

```bash
./networktest.sh --route                       # 只做回程线路检测
./networktest.sh --ping                        # 只做全国延迟测试
./networktest.sh --speed-cn --yes              # 全国测速, 跳过流量确认
./networktest.sh --speed-global --fast --yes   # 全球测速, 精简节点
./networktest.sh -a --yes --output report.txt  # 全面测试并保存报告
```

| 参数 | 说明 |
|---|---|
| （无参数） | 进入交互式菜单 |
| `--route` | 回程线路检测 |
| `--ping` | 全国延迟测试 |
| `--speed-cn` | 全国测速 |
| `--speed-global` | 全球测速 |
| `-a`, `--all` | 全面测试（依次执行全部模块） |
| `--fast` | 测速只跑精简节点集（三网各 1 个 / 每大区 1 个），大幅省时间省流量 |
| `--yes` | 跳过测速前的流量消耗二次确认 |
| `--verbose` | 线路检测时输出完整 traceroute 路径 |
| `--output FILE` | 将报告（自动去除颜色码的纯文本）保存到文件，终端仍正常显示 |
| `--ipv6` | 预留，后续版本实现 |
| `-h`, `--help` | 帮助 |

## 3. 结果解读

### 3.1 回程线路结论

示例输出：

```
==================== 回程线路结论汇总 ====================
 目标           线路                                           证据
------------------------------------------------------------------
 电信(上海)     电信 CN2 GIA (AS4809, 精品线路)                59.43.80.85
 联通(上海)     联通 4837 骨干网 (AS4837, 普通线路)            219.158.19.70
 移动(上海)     移动 CMI (AS58453, 普通线路)                   223.120.2.54
```

各线路标签的含义与质量定位：

| 标签 | 运营商 | 判定依据 | 质量 |
|---|---|---|---|
| **CN2 GIA** | 电信 | 路径含 `59.43.*`（AS4809）且无 `202.97.*` | ★★★★★ 精品，晚高峰最稳 |
| **CN2 GT** | 电信 | 同时出现 `59.43.*` 与 `202.97.*` | ★★★★ 半精品 |
| **163 骨干网** | 电信 | 仅 `202.97.*`（AS4134） | ★★ 普通，晚高峰易拥塞 |
| **联通 9929** | 联通 | `218.105.*` / `210.51.*`（AS9929） | ★★★★ 精品 |
| **联通 4837** | 联通 | `219.158.*`（AS4837） | ★★★ 普通偏好 |
| **移动 CMIN2** | 移动 | AS58807 | ★★★★ 精品 |
| **移动 CMI** | 移动 | `223.118-120.*`（AS58453） | ★★★ 普通 |
| **[途经: NTT/Cogent/...]** | — | 路径经第三方国际骨干绕行 | 视具体骨干而定，NTT/Cogent 晚高峰通常较差 |
| **未知/疑似专线** | — | 未命中任何特征 | 可能是 IPLC/IEPL 专线（路径不透明），也可能特征库需更新 |

> **重要**：脚本测的是**回程**（VPS → 大陆）。**去程**（大陆 → VPS）无法在 VPS 上直接测量，
> 建议配合大陆侧工具验证：[ITDog](https://www.itdog.cn/traceroute/)、[ping.pe](https://ping.pe/)，
> 输入你的 VPS IP 即可看三网去程路由。去程回程线路可能不一致（例如去程 163、回程 CN2）。

### 3.2 全国延迟

- 每个节点 ping 10 个包，显示平均/最小/最大延迟和丢包率；
- 平均延迟颜色：绿色 < 150ms ≤ 黄色 < 250ms ≤ 红色；
- 每个运营商末尾给出平均延迟、延迟最优与最差的城市；
- 部分节点"超时"不代表线路故障——不少大陆 IP 禁 ping，整组全部超时才需要关注。

参考基准（供判断线路好坏）：

| VPS 位置 | 到大陆正常延迟 |
|---|---|
| 香港/日本/韩国/新加坡 | 30 ~ 100 ms |
| 美国西海岸 | 130 ~ 200 ms |
| 美国东海岸 / 欧洲 | 200 ~ 300 ms |

### 3.3 测速结果

- 下载/上传单位为 **Mbps**（兆比特每秒），除以 8 约等于 MB/s；
- "实际服务器"一列显示 Speedtest 实际匹配到的节点，供核对；
- 单节点失败会显示"测试失败/节点不可用, 已跳过"，不影响其他节点。

## 4. 测速节点维护

测速节点定义在脚本顶部的 `SPEED_CN_NODES` / `SPEED_GLOBAL_NODES` 数组中，条目格式：

```
来源|标签|后备ID(逗号分隔,可空)|测试数量(可空,默认1)
```

示例：

```bash
"search:China Telecom@China;CT5G@China|电信 *|3633,5396|2"
#  │                                     │        │      └ 该运营商测 2 个不同节点
#  │                                     │        └ 搜索失败时的静态后备 server id
#  │                                     └ 标签, 末尾 * 表示 --fast 模式保留
#  └ 运行时查询官方接口, 多个关键词用 ; 分隔依次尝试, @国家 过滤海外同名节点
"nearest|就近节点 *"          # 就近自动选择
"3633|电信-上海"              # 固定 server id
```

工作机制：

- **搜索结果 + 后备 ID 组成候选池**，某个节点测试失败会静默换下一个候选，直到凑满"测试数量"，全部失败才会显示跳过；
- **`@国家` 过滤**很重要：从海外搜 "China Telecom" 可能匹配到 "China Telecom Americas"（美国节点），加 `@China` 后只保留大陆节点，避免测出假的大陆速度；
- 同一次运行中已测过的节点自动去重，不会重复测试；
- 大陆的 Speedtest 节点近年大量下线（尤其是移动），搜不到属正常现象。

全国延迟节点在 `PING_NODES` 数组，格式 `地区|运营商|IP`，同样可自行增删。部分节点"超时"通常是该 IP 禁 ping，可替换为该地区其他运营商 IP。

## 5. 流量与耗时预估

| 模块 | 耗时 | 流量 |
|---|---|---|
| 回程线路检测 | 1 ~ 2 分钟 | < 10 MB |
| 全国延迟测试 | 1 ~ 3 分钟 | < 5 MB |
| 全国测速 | 3 ~ 10 分钟 | 数百 MB ~ 数 GB |
| 全球测速 | 5 ~ 15 分钟 | 1 ~ 数 GB |

**流量计费的 VPS 请注意**：测速类功能执行前会提示确认（`--yes` 跳过）；流量紧张时用 `--fast`。

## 6. 常见问题

**Q: 提示 NextTrace 下载失败？**
脚本会自动降级为系统 `traceroute`。降级后没有 ASN 标注，线路判定仅依赖 IP 段特征，精度略降但主流线路（CN2/163/4837/9929/CMI）仍可识别。也可手动安装 NextTrace 后重跑。

**Q: 回程结论是"未知/疑似专线"？**
说明路径未命中任何骨干网特征。常见于走 IPLC/IEPL 专线的机器（中间路由不可见），或者 traceroute 大量星号。可加 `--verbose` 查看完整路径自行判断。

**Q: 全国测速所有节点都失败？**
大陆 Ookla 节点确实少且不稳定。可以改用第 3 方判断：结合"全国延迟测试"的延迟/丢包，以及从大陆侧实际下载 VPS 上的文件测试。

**Q: 三网结论为什么不一样？**
完全正常。很多套餐只对电信优化（如 CN2 GIA），联通/移动仍走普通线路。按你的宽带运营商看对应行即可。

**Q: 判定结果和商家宣传不符？**
以脚本实测为准，但注意：① 去程回程可能不同；② 部分机器高峰期切换线路；③ 特征库基于公开 IP 段，运营商调整后可能需更新脚本。
