# VPS 大陆线路检测与测速脚本 — 需求文档

- **项目名称**：NetworkTest — VPS 大陆回程线路检测与测速脚本
- **文档版本**：v0.2（草案）
- **日期**：2026-07-02
- **运行环境**：Linux（VPS 端运行，Bash 脚本）
- **v0.2 变更**：新增交互式菜单模式；全国延迟测试扩展至全国主要省市；测速拆分为"全国测速"与"全球测速"两个独立模块

---

## 1. 背景与目标

### 1.1 背景

购买海外 VPS 后，用户最关心的问题之一是：**这台 VPS 是通过什么线路/渠道接入中国大陆的**。不同线路的质量和价格差异巨大：

| 运营商 | 线路 | ASN | 特征 IP 段 | 质量定位 |
|---|---|---|---|---|
| 电信 | 163 骨干网（普通） | AS4134 | 202.97.x.x | 普通，晚高峰易拥塞 |
| 电信 | CN2 GT | AS4809 + AS4134 | 59.43.x.x（省级出现 202.97） | 半程精品 |
| 电信 | CN2 GIA | AS4809 | 全程 59.43.x.x | 精品，最优 |
| 联通 | 4837（普通） | AS4837 | 219.158.x.x | 普通偏好 |
| 联通 | 9929（CUII/CUG） | AS9929 | 218.105.x.x / 210.51.x.x | 精品 |
| 移动 | CMI（普通） | AS58453 | 223.120.x.x 等 | 普通 |
| 移动 | CMIN2（精品） | AS58807 | — | 精品 |

此外还存在 IEPL/IPLC 专线、以及经第三方中转（如 NTT、Cogent、PCCW、Telia 等国际骨干绕行）的情况。

### 1.2 目标

开发一个在 Linux VPS 上一键运行的 Bash 脚本，通过**交互式菜单选择功能**，实现：

1. **线路识别**：自动判断 VPS 到大陆三大运营商（电信/联通/移动）分别走的是哪条线路（去程与回程分别标注）。
2. **全国延迟测试**：对大陆主要省市三网节点的延迟、丢包、路由跳数进行测量。
3. **全国测速**：对大陆三网多地测速节点做上传/下载测速。
4. **全球测速**：对全球主要地区（亚太/欧洲/北美等）测速节点做上传/下载测速。
5. **报告输出**：生成一份可读性强的终端报告（可选保存为文件），给出线路结论（例如"电信回程：CN2 GIA"）。

以上功能在**同一个脚本**内实现，启动后以菜单编号选择运行哪个模块（也支持命令行参数直达，便于自动化）。

### 1.3 非目标（本期不做）

- 不做长期监控/定时巡检（后续版本可加 cron 模式）。
- 不做大陆侧发起的去程测量（去程只能通过路径推断或提示用户借助 ping.pe / ITDog 等平台验证）。
- 不提供 Web 界面。

---

## 2. 术语说明

- **去程**：大陆用户 → VPS 方向的路由（需从大陆侧测量，VPS 上只能间接推断）。
- **回程**：VPS → 大陆方向的路由（可在 VPS 上直接 traceroute 测量，**本脚本的核心**）。
- **三网**：中国电信（CT）、中国联通（CU）、中国移动（CM）。

## 3. 功能需求

### FR-0 交互式菜单（功能入口）

脚本启动后显示主菜单，用户输入编号选择功能：

```
========== NetworkTest 主菜单 ==========
 1. 回程线路检测          （判断接入大陆的线路类型）
 2. 全国延迟测试          （三网各省市 ping 延迟/丢包）
 3. 全国测速              （大陆三网多地上传/下载测速）
 4. 全球测速              （全球主要地区上传/下载测速）
 5. 全面测试              （依次执行 1-4）
 0. 退出
请输入选项 [0-5]:
```

- 输入非法时重新提示，不退出。
- 每个模块执行完毕后返回主菜单，可继续选择其他功能，直到选择退出。
- 同时支持**命令行参数直达**（见 FR-7），带参数运行时跳过菜单，便于脚本化/自动化调用。
- 测速类选项（3/4/5）在执行前提示预计流量消耗并二次确认（可用参数跳过确认）。

### FR-1 环境自检与依赖安装

- 检测系统发行版（Debian/Ubuntu/CentOS/Alma/Rocky 等）并适配包管理器（apt/yum/dnf）。
- 自动检测并安装依赖：`curl`、`traceroute`/`mtr`、`bc`、`jq`。
- 自动下载路由追踪增强工具 **NextTrace**（优先，带 ASN/地理位置标注），失败时降级为系统 `traceroute`/`mtr`。
- 检测 root 权限（ICMP 原始套接字需要），非 root 时给出提示并尽量降级运行。
- 支持 IPv4 为主，IPv6 可选（`--ipv6` 参数）。

### FR-2 VPS 基本信息采集

- 出口公网 IP、所属 ASN、机房/地理位置（调用 ip-api.com / ipinfo.io 等，多源容错）。
- CPU / 内存 / 虚拟化类型（可选，简要展示）。
- TCP 拥塞控制算法（是否启用 BBR）。

### FR-3 全国延迟测试（菜单项 2）

- 对预置的三网**全国主要省市**节点执行 ping，覆盖范围：
  - **核心城市（必测）**：北京、上海、广州/深圳、成都 × 电信/联通/移动（≥12 个目标）；
  - **扩展省市（尽量覆盖）**：杭州、南京、武汉、郑州、西安、重庆、天津、沈阳、哈尔滨、济南、福州/厦门、长沙、昆明、乌鲁木齐 等 × 三网，总目标数 30~50 个。
- 每目标 ≥10 个包，输出最小/平均/最大延迟与丢包率；多目标**并发执行**以控制总耗时。
- 结果按运营商分组展示，并给出每个运营商的平均延迟汇总，标注延迟最优/最差地区。
- 目标节点表内置于脚本中，需可维护、可替换（集中定义在数组/配置段，格式：`地区|运营商|IP`）。
- 参考目标示例（需在实现时验证可用性）：
  - 电信：广州 58.60.188.222 类、上海 202.96.209.133、北京 219.141.136.12
  - 联通：广州 210.21.196.6、上海 210.22.97.1、北京 202.106.50.1
  - 移动：广州 120.196.165.24、上海 211.136.112.200、北京 221.179.155.161

### FR-4 回程线路识别（菜单项 1，核心功能）

- 对三网各选 1~3 个代表节点执行 traceroute（优先 NextTrace，TCP/ICMP 模式可选）。
- 解析路径中每一跳的 IP，与内置特征库匹配，判定线路类型：
  - 出现 `59.43.*` 且骨干段无 `202.97.*` → **CN2 GIA**
  - 同时出现 `59.43.*` 与 `202.97.*` → **CN2 GT**
  - 仅 `202.97.*` → **电信 163**
  - 出现 `218.105.*` / `210.51.*`（AS9929）→ **联通 9929**
  - 仅 `219.158.*`（AS4837）→ **联通 4837**
  - 出现 AS58807 特征 → **CMIN2**；仅 AS58453 → **移动 CMI**
- 特征库以"IP 前缀 → 线路标签"映射表形式集中维护，便于更新。
- 每个方向输出：**结论标签 + 关键证据跳（IP/ASN）+ 完整路径（可选 `--verbose`）**。
- 若路径经第三方国际骨干（NTT/Cogent/PCCW/Telia/GTT 等），识别并标注"绕行 via XXX"。
- 明确在报告中注明：此为**回程**结果；去程需大陆侧验证，并给出建议方法。

### FR-5 全国测速（菜单项 3）

- 集成 Speedtest（官方 CLI 或 speedtest-cli），对**大陆三网多地**测速节点测试：
  - 覆盖：电信/联通/移动 × 华北（北京/天津）、华东（上海/杭州/南京）、华南（广州/深圳）、西南（成都/重庆）等，目标 9~15 个节点；
  - 内置 server ID 列表，集中维护（格式：`server_id|运营商|城市`）；节点失效时自动跳过并提示。
- 输出：每节点的下载/上传带宽（Mbps）、测速延迟、抖动、丢包；末尾按运营商汇总平均值。
- 运行前提示预计流量消耗（全国测速一轮可能消耗数 GB 流量），支持 `--fast` 模式只测三网各 1 个节点。

### FR-6 全球测速（菜单项 4）

- 对**全球主要地区**测速节点测试，覆盖：
  - 亚太：香港、台湾、东京、大阪、新加坡、首尔、悉尼；
  - 欧洲：伦敦、法兰克福、巴黎、阿姆斯特丹；
  - 北美：洛杉矶、圣何塞、西雅图、纽约；
  - 其他：孟买、迪拜、圣保罗（可选）。
- 另包含"就近节点"自动测速（Speedtest 默认选择），作为 VPS 本机带宽基准。
- 输出格式与全国测速一致，按地区分组展示。
- 节点列表集中维护，失效自动跳过；支持 `--fast` 模式只测每大区 1 个代表节点。

### FR-7 报告输出

- 终端彩色输出，分区块：基本信息 → 回程线路结论 → 全国延迟 → 全国测速 → 全球测速（按实际执行的模块输出）。
- 核心结论汇总表，例如：

  ```
  ============ 回程线路结论 ============
  电信(上海)   CN2 GIA   (证据: 59.43.80.85 AS4809)
  联通(上海)   联通4837  (证据: 219.158.19.70 AS4837)
  移动(上海)   移动CMI   (证据: 223.120.2.54 AS58453)
  ```

- 支持 `--output <file>` 将纯文本报告保存到文件。
- 运行结束显示总耗时。

### FR-8 命令行参数（菜单直达，用于自动化）

| 参数 | 说明 |
|---|---|
| 无参数 | 进入交互式主菜单（FR-0） |
| `--route` | 直接执行回程线路检测（菜单 1） |
| `--ping` | 直接执行全国延迟测试（菜单 2） |
| `--speed-cn` | 直接执行全国测速（菜单 3） |
| `--speed-global` | 直接执行全球测速（菜单 4） |
| `-a` / `--all` | 全面测试（菜单 5） |
| `--fast` | 测速使用精简节点集 |
| `--yes` | 跳过流量消耗二次确认 |
| `--ipv6` | 附加 IPv6 测试 |
| `--verbose` | 输出完整 traceroute 路径 |
| `--output FILE` | 保存报告 |

## 4. 非功能需求

- **NFR-1 一键运行**：`bash <(curl -sL <url>)` 即可执行；无参数进入菜单交互，带参数（FR-8）可完全无交互跑完，适合自动化。
- **NFR-2 兼容性**：支持 Debian 10+/Ubuntu 20.04+/CentOS 7+ 及主流衍生版；amd64 与 arm64。
- **NFR-3 健壮性**：单个目标节点不可达/测速节点失效不得中断整体流程，应跳过并标注。
- **NFR-4 性能**：`--route` ≤ 2 分钟；`--ping`（全国延迟，并发执行）≤ 5 分钟；全国/全球测速单模块 ≤ 15 分钟（`--fast` 模式 ≤ 5 分钟）；全面测试 ≤ 35 分钟。
- **NFR-5 流量成本**：非测速模块流量消耗 < 10 MB；全国/全球测速一轮可能消耗数 GB，执行前必须提示并确认（`--yes` 可跳过）。
- **NFR-6 可维护性**：目标节点表、线路特征库、测速节点表集中定义，改动无需读懂主逻辑。
- **NFR-7 安全**：不采集/上传用户数据；下载第三方二进制（NextTrace 等）时校验来源（官方 GitHub Release），不执行来路不明代码。

## 5. 技术方案要点

- **语言**：Bash（主体），尽量避免额外运行时依赖。
- **路由追踪**：NextTrace（自带 ASN/地理标注，输出可解析）→ 降级 mtr → 降级 traceroute。
- **线路判定**：基于"骨干段特征 IP 前缀 + ASN"规则引擎（grep/awk 实现），规则集中维护。
- **测速**：Ookla speedtest CLI（注意首次运行需接受许可，脚本自动处理）。
- **已知同类工具参考**（用于校验结论正确性）：NextTrace、backtrace（回程检测）、ZBench、SuperSpeed。

## 6. 风险与限制

| 风险 | 说明 | 缓解 |
|---|---|---|
| 目标 IP 失效 | 大陆节点 IP 可能变更或禁 ping | 多节点冗余、集中维护、失败跳过 |
| 回程 ≠ 去程 | VPS 上无法直接测去程 | 报告中明确标注，提示用 ITDog/ping.pe 验证去程 |
| 特征库过时 | 运营商 IP 段调整 | 规则表化，版本化维护 |
| 测速节点限速/失效 | 大陆 Speedtest 节点常下线 | 节点列表可更新，失效自动跳过 |
| 中转/专线误判 | IPLC 等专线路径不透明 | 无法匹配特征时输出"未知/疑似专线"而非硬判 |

## 8. 验收标准

1. 在一台已知线路的 VPS（如确认为 CN2 GIA 的机器）上运行，脚本判定结论与已知事实一致。
2. 在 Debian/Ubuntu/CentOS 三类系统上均能一键跑通。
3. 菜单 1-5 各选项均可独立运行并正确返回主菜单；非法输入不导致退出。
4. 任一目标节点不可达/测速节点失效时流程不中断。
5. `--route` 模式 2 分钟内出结论；全国延迟测试覆盖 ≥12 个核心目标且并发执行。
6. 全国测速与全球测速分组汇总输出正确，`--fast` 模式生效。
